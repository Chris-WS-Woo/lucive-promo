>                                                              ↓
>                                                         총 실참여자
> ```
>
> **갱신일:** 2026-04-17 · **목표:** 실참여자 **최소 100명 · 권장 200명** · **예산:** 130-230만원

### 2-2. A-SAM (불만율 × pain 매칭) — 리뷰 편향 보정

**⚠️ 리뷰 편향 (J-curve) 보정 필요:**
- 리뷰는 **매우 불만/매우 만족한 유저**가 쓰는 경향 (중간층 silent)
- 1★+2★ 리뷰 비율은 실제 불만율의 **1.5-2배 과대추정** (J-curve effect)
- 업계 연구: 리뷰 작성 유저는 전체 유저의 1-3%, 그중 부정 비중 2-3배 편향
- 참조: [Nielsen Norman Group · "Review Bias Studies" 2022](https://www.nngroup.com/articles/review-sampling-bias/)

**보정된 fatigued 비율:**
```
리뷰 1★+2★ 평균:  26%
÷ review bias factor (1.8x)
= 실제 불만 유저 비율: ~14-15%
+ silent churners (별점 안 남기고 떠난 유저): ~10-12%
──────────────────────────────────
실제 fatigued 비율: ~24-27% (기존 45% → 보정)
```

**보정된 A-SAM 계산:**
```
A-TAM 150만
× 25% (보정된 fatigued 비율)
× 80% (LUCIVE 3대 pain 매칭: 기억·반복·서사)
= A-SAM 30만명  (기존 54만 → 보정 후 30만)
```

> **변경 영향:** SAM 44% 감소 → 상위 퍼널에서 effective impressions/CTR이 낮아질 수 있음. Section 7 시나리오 재검토 필요.

### 2-3. A 페르소나 특성

- **고의도 · 고전환**: 이미 pain 경험 → 대안 찾는 중
- **공감 톤**에 민감 ("어제 얘기 기억 못 함" 류 커뮤니티 스타일)
- **CTR · CVR 모두 페르소나 B보다 높은 편** (벤치마크: 1.3-1.5배)

## B.2 LUCIVE 적용 근거

### B.2.1 왜 Meta CTR 2.5% 적용?

- Meta 평균 3%, LUCIVE는 **신규 브랜드 + 다크 테마 크리에이티브** → 살짝 보수적 2.5%
- 페르소나 B는 매칭 낮아 2%
- 예상 CTR이 실제 1.5-2%로 나오면 **즉시 크리에이티브 교체**

### B.2.2 왜 Landing CVR 8% 적용?

- Unbounce 상위 25% = 5-11%, 중앙값 2.35%
- LUCIVE 랜딩 품질 (OTP + 비디오 + 매트릭스 + 플로팅 내비)이 상위 25% 수준
- **A 페르소나 pain 매칭 시 8-10%, B 페르소나 6-7% 차이**

### B.2.3 왜 OTP 완료율 65%?

- SendGrid 평균 65% (델리버리 + 열람 + 링크 클릭)
- LUCIVE 6자리 코드 방식 → 매직링크 대비 **완료율 +5-10%**
- 단, 스팸함 분류 리스크 감안해서 65% 유지

### B.2.4 왜 Referral K=0.3-0.4?

- Dropbox 초기 K=0.35 (OpenView)
- Airbnb 초기 K=0.25
- LUCIVE는 **2-5위 친구 3개월 무료** + 전체 특가 → 친구도 혜택 있음
- 비슷한 dual-reward 사례 K=0.4-0.6 (상위 수준)
- **보수적으로 K=0.3 (Plan B 시)**, 잘 되면 0.5까지 기대
